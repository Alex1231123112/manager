# Два способа решения проблемы входа в админку

## Проблема
В логах: `Login failed for user: admin (BadCredentialsException)` — пароль не совпадает, т.к. в контейнер могли не попадать переменные `ADMIN_USERNAME`/`ADMIN_PASSWORD` или подставлялось пустое значение.

---

## Способ 1: Явно задать логин/пароль в docker-compose
```yaml
environment:
  ADMIN_USERNAME: admin
  ADMIN_PASSWORD: admin
```
**Плюсы:** Гарантированно работает во всех окружениях.  
**Минусы:** Пароль в открытом виде в файле, который часто попадает в git. Небезопасно для продакшена.

---

## Способ 2 (выбран): Не передавать ADMIN_* из compose — использовать только дефолты Spring
В docker-compose **не** задавать `ADMIN_USERNAME` и `ADMIN_PASSWORD`. Тогда в контейнере этих переменных нет, и Spring при чтении `application.yml` подставляет значения по умолчанию из плейсхолдеров: `${ADMIN_USERNAME:admin}` и `${ADMIN_PASSWORD:admin}` → получается admin/admin.

**Плюсы:** Пароль не хранится в compose, не светится в репозитории. Для продакшена пароль задаётся через `.env` (в .gitignore) и передаётся в контейнер при необходимости.  
**Минусы:** Зависит от того, что в контейнер не попадают пустые переменные (если где-то задать `ADMIN_PASSWORD=`, будет пустой пароль).

**Для продакшена:** В `.env` задать `ADMIN_PASSWORD=надёжный_пароль` и в docker-compose добавить в секцию `app` → `environment`: `ADMIN_PASSWORD: ${ADMIN_PASSWORD}`. Тогда используется пароль из .env.
