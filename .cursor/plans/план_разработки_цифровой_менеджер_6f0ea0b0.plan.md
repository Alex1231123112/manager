### 2. Создание ROADMAP.md

Новый файл в корне: `c:\Manager\ROADMAP.md`.

**Структура ROADMAP**

- **Сверху:** блок «Ближайшие шаги» (первые 3 дня из текущего плана) — чек-листы без дублирования деталей.
- **Фазы:** 4 фазы по 2 недели (Фаза 1: MVP Core, Фаза 2: Генерация контента, Фаза 3: Управление командой, Фаза 4: Монетизация и упаковка) с одной целью на фазу и 5–7 ключевыми результатами (чеками).
- **Срок:** 8 недель + примечание «рекомендуемый буфер: +1–2 недели на фазы 2 и 4».
- Без таблиц технологий и без полного SQL — только вехи и чек-листы для трекинга. Ссылка внизу: «Подробности: План разработки».

Пример начала ROADMAP:

```markdown
# ROADMAP: Цифровой менеджер

## Ближайшие шаги
- [ ] Репозиторий GitHub, бот в BotFather, Spring Boot проект
- [ ] PostgreSQL, /start отвечает "Привет!", деплой на сервер
- [ ] Модели Team/Player, сохранение команды в БД, тест с друзьями

## Фаза 1 (недели 1–2): MVP Core
...
```
