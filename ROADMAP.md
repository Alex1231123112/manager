# ROADMAP: Цифровой менеджер

Срок: 8 недель. Рекомендуемый буфер: +1–2 недели на фазы 2 и 4.

---

## Ближайшие шаги

- [ ] Создать репозиторий на GitHub, зарегистрировать бота в BotFather, создать пустой Spring Boot проект
- [ ] Настроить подключение к PostgreSQL, сделать команду /start с ответом «Привет!», залить на сервер и проверить
- [ ] Создать модели Team и Player, научить бота сохранять команду в БД, позвать друзей протестировать

---

## Фаза 1 (недели 1–2): MVP Core

**Цель:** Рабочий бот с базовым функционалом

- [x] Проект настроен: Spring Boot, PostgreSQL, Telegram Bot API, структура пакетов
- [x] JPA-сущности и репозитории (Team, Player, Match), миграции, фильтрация по team_id
- [x] Команда /start, добавление игроков, просмотр состава, простое меню (кнопки)
- [x] Создание матча, ввод результата, сохранение в БД, вывод результатов
- [x] Результат: бот создаёт команду, добавляет игроков и сохраняет результаты матчей

---

## Фаза 2 (недели 3–4): Генерация контента

**Цель:** Автоматическое создание постов и картинок

- [x] Система шаблонов текста с подстановкой переменных (после матча — пост в чат)
- [x] Карточка результата 1080×1080 PNG (Graphics2D), отправка в чат после /result
- [x] После ввода результата — генерация картинки и отправка в Telegram
- [x] Кнопка «Опубликовать в канал»
- [ ] Тест на своей команде, исправление критических ошибок
- [x] Результат: после ввода счёта бот присылает готовые картинки для соцсетей

---

## Фаза 3 (недели 5–6): Управление командой

**Цель:** Автоматизация администрирования

- [x] Опросы на игру (/poll текст), варианты Еду / Не еду / Опоздаю
- [x] Финансовый учёт: долг у игрока, /debt список, /setdebt Имя Сумма
- [x] Статусы игроков (активен, травма, отпуск, не оплатил), фильтрация состава
- [x] Планировщик напоминаний: за 24 ч и за 3 ч до матча, после матча — запрос статистики
- [x] Результат: бот напоминает, собирает явку и следит за долгами

---

## Фаза 4 (недели 7–8): Упаковка и запуск

**Цель:** Готовый к эксплуатации продукт (без платёжных систем)

- [ ] Мультитенантность: разделение по командам, роли (админ, капитан, игрок)
- [ ] Возможность отмечать оплату (кнопка/команда «оплатил», сброс долга — без ЮKassa/CloudPayments)
- [ ] Админ-панель (Web): дашборд, статистика, редактирование игроков
- [ ] Нагрузочное тестирование, проверка доступа, бэкапы PostgreSQL
- [ ] Документация для пользователей, подготовка к запуску
- [ ] Результат: готовый продукт
